1. What is mobile application testing?

Mobile application testing is responsible for validating that the system operates correctly and that there are no faults that interfere with its operation.

2. Types of mobile applications

2.1 Native applications

Native applications are those that are developed specifically for a particular operating system, such as iOS or Android.

These applications are usually faster and more efficient than other options, as they are specifically designed to take full advantage of the operating system's features..

2.2 Web Apps

Progressive web apps are a good option if you want to have an app that feels like a native app, but don't want to develop a platform-specific app.

These apps use web technologies such as HTML, CSS, and JavaScript, but can access device features such as camera and location.

2.3 Hybrid applications

Hybrid applications are a combination of web and native applications, they are a good option if you want to develop an application for multiple platforms without having to write specific code for each operating system.

These applications are developed using web technologies, such as HTML, CSS and JavaScript, and then wrapped in a native container that allows them to access device features.

3. Importance of mobile testing

Security tests seek to validate the application's resistance to attacks by malicious users, in addition to validating that developers apply security practices when programming.

4. Types of mobile application testing

What is mobile application testing?

Mobile application testing is responsible for validating that the system operates correctly and that there are no faults that interfere with its operation.

4.1 Functional Testing

Functional testing seeks to ensure that the mobile application works as indicated in the requirements and/or user stories. Some of the main functionalities to test are:

Registration/Login

- Verify that the user can register and login.
- Verify integration with applications such as Facebook, Twitter, Snapchat or Google.

Registration/Login

- Verify if the App has landscape option enabled.

4.2 Compatibility testing

For compatibility testing it is advisable to create a combined device testing strategy (mix of physical and real devices). Regardless of the type of application, the design must be responsive.

Like desktop apps, mobile web apps should be tested on various browsers, such as Google Chrome, Safari, Firefox, Edge, Opera and Samsung. The operating system versions supported by the mobile application must be defined.

4.3 Usability Testing

The goal of usability testing is to ensure that an intuitive and easy-to-use mobile application is built.

Our role as testers is to help coordinate and guide users to get the best possible feedback to determine what decisions to make regarding the app, and whether it is feasible to release it to the market or execute some changes before release.

4.4 Performance tests

Performance tests have the function of avoiding defects by subjecting the system to extreme or unusual usage loads. In this way it will be known if the application will be able to withstand them in a real situation.

4.5 Security Testing

It is done to verify whether the application data is secure or not and avoid vulnerabilities to protect it from data and privacy related threats.

4.6 Installation Tests

These tests help to verify and validate that the application is installed properly with new installations and upgrades on a periodic basis.
