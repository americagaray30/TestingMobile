5. When to use automated tests and when to use manual tests?

It's about taking a holistic approach to raising software quality standards.

5.1 automation tests:

They are very important to speed up the testing process with high accuracy and reliability, when it comes to functionality and first impressions, manual testing takes first place. Even if automated testing gets more refined.

5.2 Manual tests:

They will continue to play a role in launching quality-based applications.

It is best for exploratory testing, usability testing, and ad-hoc testing.

6. Tips for effective mobile testing.

Here are some mobile testing best practices to keep in mind when testing mobile apps:

- Use test automation to increase efficiency and productivity.
- Test early and often by using tests as part of your app development.
- Divide your app tests into smaller units.
- Make your tests data driven.
- Don’t ignore the object-oriented approach in your app testing.
- Commit your efforts towards performance and load testing.
- Distribute the testing efforts across the entire team members including developers.
- Include experts or experienced people in your QA team.
- Know the platform’s (Android or iOS) user interface/user experience (UI/UX) guidelines before starting the mobile app testing.
- Don’t just rely on automation testing as some parts of the testing require manual efforts.

7.Process for testing a mobile application.

Here is the step-by-step process for testing a mobile app:

7.1 Planning:

In the planning phase, you figure out what you are trying to achieve and the current constraints.

7.2 Identification of test types:

Before you start testing, it's crucial to identify the test types you need to test your specific mobile app.

7.3 Script design and test case:

In this phase, you prepare a test case document for all features and functionality.

7.4 Control and summary:

This phase includes detailed and comprehensive testing to give the application more stability and to eliminate minor defects.

8. Main Mobile testing tools

8.1 Android:

With the increase in Android app development, the need for mobile app testing also increases. Some of the best Android based app testing tools are:

Appium – is a common framework for creating automated tests for Android and iOS
Testsigma – is a unified test automation platform that enables easy automation for desktop, mobile (both iOS and Android), and API tests from the same place without any additional setup or configuration expense.
Calabash – is an automation testing framework for performing acceptance testing for Android and iOS applications.
UI Automator – has the ability to interface with all kinds of Android software products.
Robotium – is a free Android UI testing tool that is suitable for test automation of different versions and subversions of Android.
Ranorex – Ranorex is a good tool for test automation and is known for its detailed reports including screenshots.
